Git Flow

常用 Git 工作流，不堆无关命令。

Daily Flow

检查git status
新分支git switch -c feature/name
提交git add . && git commit -m "feat: update"
推送git push origin main

Cleanup

撤销暂存git restore --staged .
回退提交git reset --soft HEAD~1
看日志git log --oneline --graph -10
检查空白git diff --check

Release

打标签git tag v1.0.0
推标签git push origin v1.0.0
同步git pull --ff-only
远端git remote -v

Release Checks

发布后最少要确认这些点。

Content

  • 标题和描述符合分类。
  • 链接都是开发/部署/文档相关。
  • 没有混入生活导航或后台入口。

Mobile

  • 不横向溢出。
  • 卡片不压字。
  • 按钮足够大，能单手点击。

Network

  • HTTP 返回 200。
  • CSS/JS/图片无 404。
  • HTTPS 证书签发后再强制开启。

Scope

这个域名的边界。

  • 放：Git、代码、开发工具、调试工具、部署文档、发布检查。
  • 不放：个人生活导航、媒体娱乐、后台管理、账号敏感入口。
  • 原则：静态、公开、轻量、可长期保活。
